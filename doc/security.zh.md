# 安全策略

**简体中文** | [English](./security.en.md)

## 支持的版本

只支持 `main` 分支。请跑最新发布的镜像，或者从 `main` 重新构建。

## 威胁模型

这个服务会上传音频、运行说话人分离、存储说话人声纹。它的定位是**可信部署环境**。
默认情况下，任何能访问 `:8780` 的客户端都可以拿到：

- `data/uploads/` 下所有上传过的音频文件
- `data/transcriptions/` 下所有转录结果
- 所有已登记的声纹（持久化的说话人 embedding）

请把这个服务当成一个内部数据库对待。

## 必须做的硬化

1. **一定要设 `API_KEY`**。不设的话服务会无认证接收所有请求，并在启动时打 warning。
   任何不是纯内网可信网段的部署都 **必须** 把这个环境变量设为一串长随机字符串。
   客户端通过 `Authorization: Bearer <key>` 或 `X-API-Key: <key>` 带上它。
2. **`.env` 永远不要提交到 git**。仓库里只该有 `.env.example`。
3. **不要把 `:8780` 直接暴露到公网**。请挂在 VPN 后面，或加一层带 TLS 的反向代理，
   或者至少加 IP 白名单。`API_KEY` 单独用不能替代传输加密。
4. **HuggingFace token 不要进日志、不要打进镜像**。它只在运行时通过 `HF_TOKEN`
   被读一次，用来下载 pyannote 模型，除此之外无其他用途。
5. **请备份 `data/voiceprints/`**。丢了就得重新登记每个人。

## 漏洞上报

请在 GitHub 提交 private security advisory，或者发邮件给仓库维护者。**不要** 在公开
issue 里提未修复的漏洞。
