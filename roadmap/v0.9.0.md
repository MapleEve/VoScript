# v0.9.0 Planning

## 版本定位

v0.9.0 是从“技术功能可用”走向“产品协议稳定”的关键阶段。

## 主目标

### 1. batch / job 语义稳定
- 统一提交、轮询、完成、失败、恢复、去重等作业语义
- 让接入方可以把 VoScript 当成稳定后端，而不是一次性脚本服务

### 2. transcript 语义稳定
- 让 transcript 不只是结果文件，而是具备明确语义边界的产品对象
- 梳理 segments、speaker_map、unique_speakers、人工修正结果之间的关系
- 为 post-1.0 transcript state machine 留出最小兼容路径

### 3. speaker identity 语义稳定
- 保持 speaker_label、speaker_name、speaker_id 的角色明确
- 保持持久化声纹与单次录音内标签之间的清晰映射关系
- 避免因为新增能力而使旧调用方语义混乱

### 4. product API 视角稳定
- 继续以 HTTP API 为第一接入面
- 让 BetterAINote 与其它调用方都能基于同一套稳定心智接入

## 不在本版本承诺的内容

- 不把 streaming 作为必须完整落地的 1.0 前硬承诺
- 不把全语音任务统一平台作为 v0.9 的目标

## 完成标准

- 文档能够清楚说明 job / transcript / speaker identity 的稳定语义
- 兼容性原则能映射到实际 API 和持久化对象
- v1.0 的定义可以建立在 v0.9 的协议稳定基础上
