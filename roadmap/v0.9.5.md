# v0.9.5 Planning

## 版本定位

v0.9.5 是 1.0 前的冻结预演阶段，核心不是再发明新范围，而是验证“我们是否真的准备好把现有能力作为稳定产品承诺”。

## 主目标

### 1. 兼容性冻结预演
- 对照 compatibility map 逐项检查
- 找出仍然可能导致升级惊喜的隐式行为
- 把破坏性变化在 1.0 前清理掉或明确延后

### 2. upgrade / migration posture 固化
- 明确已有部署如何升级
- 明确已有 result.json / status.json / voiceprints.db 的迁移原则
- 让 1.0 发布不是一次“用户自己猜”的升级

### 3. 回归矩阵与文档对齐
- 让 README、快速上手、API 文档、roadmap 叙事一致
- 对外只保留已经准备承诺的范围

## 关键问题

- 是否还存在会破坏旧客户端心智的状态语义变化
- 是否还存在会破坏旧数据读取的持久化变化
- 是否还存在“长期 vision 看起来像当前承诺”的文案歧义

## 完成标准

- 1.0 范围清晰可数
- roadmap、README、compatibility map 三者表述一致
- 剩余 post-1.0 项目都被显式留在 vision 文档，而不是夹带进 release scope
