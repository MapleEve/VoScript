# v0.8.0 Planning

## 版本定位

v0.8.0 的职责不是把能力面无限铺开，而是把 1.0 前需要长期稳定的架构边界整理清楚。

## 主目标

### 1. 管线形状稳定化
- 按 canonical pipeline shape 统一阶段认知
- 让 stage slot 成为稳定架构概念，而不是实现细节
- 为未来多后端替换打边界

### 2. provider 插拔边界稳定化
- 把“模型/后端可替换”与“产品语义稳定”拆开
- provider 负责推理实现差异
- transcript、speaker identity、持久化语义仍由产品层与领域层主导

### 3. artifact / persistence 责任分界
- 明确 job status、final result、voiceprint storage 的责任面
- 降低后续 batch / streaming 统一时的破坏性调整风险

## 规划中的结果

- 未来可以替换 ASR / VAD / diarization / enhancement 等后端
- 但替换后端不等于把项目改造成通用 runtime
- 1.0 前仍然以 meeting speech backend 的产品边界为主

## 风险控制

### 要避免的误区
- 为了“支持更多后端”而提前扩大产品叙事
- 把 provider 结构设计成用户直接面对的复杂能力矩阵
- 把 post-1.0 vision 的大词提前下压成 v0.8 的执行承诺

## 完成标准

- 架构文档能清楚区分 stable slot、provider、domain policy、infra
- compatibility map 中列出的对象在 v0.8 后更不容易被破坏式修改
- 为 v0.9 的产品协议稳定化打好边界基础
