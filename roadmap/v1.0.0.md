# v1.0.0 Planning

## 1.0 的定义

VoScript 1.0 的意义不是“什么都做”，而是“核心产品定义稳定成立”。

在 1.0，VoScript 仍然首先是：

- 自托管 GPU 转录服务
- 持久化声纹系统
- 面向产品接入的 HTTP API

## 1.0 应该稳定下来的东西

### 1. 产品叙事稳定
- 外部理解应该保持一致：VoScript 是会议/录音场景的语音应用后端基础版本
- 不因为 vision 扩大而改变 1.0 的基本产品定义

### 2. API 与结果协议稳定
- `status.json`、`result.json` 的核心读取心智稳定
- job 查询、结果获取、speaker 相关字段语义稳定
- 接入方可以据此长期集成

### 3. speaker memory baseline 稳定
- 虽然完整 speaker memory 是 post-1.0 方向，但 1.0 要先把持久化声纹的基础产品语义稳定下来
- `voiceprints.db` 的角色、迁移原则、数据安全预期要清楚

### 4. 扩展边界稳定
- 可以继续有不同推理后端
- 但 1.0 不把自己定义成通用声学 runtime
- 后续 vision 扩大以 product backend 为中心推进

## 1.0 不代表什么

1.0 不代表：

- 已经完成 batch / streaming 全统一
- 已经成为通用语音基础设施
- 已经覆盖所有语音任务
- 已经变成另一个 FunASR 或某个现成产品的复刻

## 1.0 之后怎么走

1.0 之后，VoScript 可以继续扩大，但扩大方向以 [vision-post-1.0.md](./vision-post-1.0.md) 为准：

- 可插拔推理后端
- batch / streaming 协议统一
- speaker memory
- transcript state machine
- orchestration / product API

这些方向属于 1.0 之后的 horizon，不倒灌回 1.0 的发布承诺。
